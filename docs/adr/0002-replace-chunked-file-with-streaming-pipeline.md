# Replace Chunked-File with Streaming Pipeline

The packaged server will replace `ChunkedFilePipeline` and the `CORO_PIPELINE=chunked-file` selector with a `Streaming Pipeline` that pipes request audio directly into ffmpeg, consumes decoded PCM as sequential chunks, feeds incremental ASR windowing and online diarization, and never materializes the full upload or decoded PCM in RAM. `FullMemoryPipeline` remains as the simple baseline, while `chunked-file` is removed rather than aliased because its current behavior writes request audio to a temp file and still rejoins decoded PCM into a full buffer, which does not provide the bounded-memory/no-disk property the selector name implies.

The public `/v1/audio/transcriptions` JSON and SSE contracts remain unchanged: SSE may emit transcript deltas as ASR windows complete, and speaker assignment remains part of final response construction after diarization finishes. Incremental ASR windowing preserves the existing 30-second window and 2-second overlap semantics so the rewrite changes memory and IO characteristics without intentionally retuning transcription behavior.
